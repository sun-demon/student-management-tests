# Пошаговая инструкция по TestIT

Инструкция для итоговой работы: ручные тест-кейсы, прогон, автотесты.

---

## Шаг 0. Регистрация

1. Откройте https://id.testit.software/registration
2. Выберите **бесплатную облачную** регистрацию (Lite)
3. Подтвердите email и войдите в систему

Ваш URL будет вида: `https://ВАШ_АККАУНТ.testit.software`

---

## Шаг 1. Создайте проект

1. В левом меню: **Проекты** → **Создать проект**
2. Название: `Student Management` (или как в отчёте)
3. Сохраните

**Скопируйте Project ID:**
- Проекты → меню (⋮) у проекта → **Скопировать ID**
- Понадобится для `connection_config.ini` и GitHub Secrets

---

## Шаг 2. Создайте конфигурацию (для автотестов)

1. **Настройки проекта** → **Конфигурации** → **Создать**
2. Название: `Python + Chrome CI` или `Local pytest`
3. Скопируйте **GUID конфигурации**

---

## Шаг 3. Ручные тест-кейсы (раздел 3.1 отчёта)

1. Откройте проект → **Тест-кейсы**
2. Создайте секцию: `UI — Ручное тестирование`
3. Добавьте тест-кейсы по функционалу:

| ID (в TestIT) | Название | Что проверить |
|---------------|----------|---------------|
| TC-UI-001 | Успешная регистрация | POST /register или форма /register |
| TC-UI-002 | Успешная авторизация | /login → редирект на /add-user |
| TC-UI-003 | Негативный возраст | Возраст -5 → ошибка |
| TC-UI-004 | Добавление студента | Валидные данные → успех |
| TC-UI-005 | Невалидная дата | Неверный формат даты |
| TC-UI-006 | ФИО со спецсимволами | О'Коннор-Смит |
| TC-UI-007 | Список студентов | /users-page |
| TC-UI-008 | Повторная отправка формы | Кнопка блокируется (на улучшенном стенде) |
| TC-UI-009 | Выход из системы | /logout-ui |
| TC-UI-010 | Удаление студента | Подтверждение в модальном окне |

### Как оформить один тест-кейс

1. **Создать тест-кейс**
2. **Предусловия:** «Администратор зарегистрирован и авторизован»
3. **Шаги:**
   - Перейти на `/add-user`
   - Заполнить поля...
   - Нажать «Добавить»
4. **Ожидаемый результат:** «Студент отображается в списке»
5. **Приоритет:** Medium / High

---

## Шаг 4. Ручной тестовый прогон

1. **Прогоны** → **Создать прогон**
2. Название: `Ручное UI тестирование — Демин Д.П.`
3. Выберите созданные тест-кейсы
4. **Запустить прогон**

### Проставление результатов

Для каждого теста в прогоне:
- **Passed** — если работает как ожидается
- **Failed** — если баг (опишите в комментарии)
- **Blocked** — если стенд недоступен

> **Важно (Lite):** экспорт отчёта прогона может быть недоступен.  
> **Сделайте скриншоты** экрана прогона с результатами и вложите в архив + раздел 3.1 отчёта.

## Шаг 5. Тестирование на вашем VPS

После деплоя mock-сервера:

```bash
# На VPS
cd /opt/edumanage
docker compose up -d
```

Откройте `http://ВАШ_IP:5000` и пройдите сценарии из тест-кейсов вручную, проставляя результаты в TestIT.

В отчёте укажите URL вашего стенда вместо `158.160.87.146:5000`.

## Шаг 6. Интеграция автотестов (pytest → TestIT)

### 6.1 Получите токен

1. Профиль (аватар) → **Настройки** → **Безопасность**
2. **Создать приватный токен** → скопируйте

### 6.2 Настройте проект

```bash
cd student-management-tests
cp connection_config.ini.example connection_config.ini
```

Заполните:

```ini
[testit]
url = https://ВАШ_АККАУНТ.testit.software
privateToken = ВАШ_ТОКЕН
projectId = GUID_ПРОЕКТА
configurationId = GUID_КОНФИГУРАЦИИ
adapterMode = 2
testRunName = Student Management Autotests
certValidation = false
automaticCreationTestCases = true
```

### 6.3 Запустите автотесты

```bash
source .venv/bin/activate
pytest tests/ --testit
```

После прогона в TestIT появится новый **прогон автотестов** с результатами.

### 6.4 Альтернатива без адаптера (JUnit XML)

```bash
pytest tests/ --junitxml=reports/results.xml
pip install testit-cli
export TMS_TOKEN=ваш_токен
testit results import \
  --url https://ВАШ_АККАУНТ.testit.software \
  --project-id GUID_ПРОЕКТА \
  --configuration-id GUID_КОНФИГУРАЦИИ \
  --testrun-name "Local import" \
  --results reports
```

Скопируйте `reports/results.xml` → `testit_results.xml` для архива.

## Шаг 8. Связь ручных и автотестов

В коде автотестов уже есть декораторы:

```python
@testit.externalId("UI-AUTO-001")
@testit.workItemIds("TC-UI-001")
```

`workItemIds` связывает автотест с ручным тест-кейсом в TestIT (если ID совпадают).

## Частые вопросы

**Нужно ли подделывать xlsx?**  
Нет. В задании разрешены скриншоты прогона для Lite.

**Mock или оригинальный стенд?**  
Для сдачи достаточно вашего VPS со стендом. В отчёте честно укажите URL.

**TestIT сложный?**  
Для сдачи достаточно: 8–10 тест-кейсов + 1 ручной прогон + 1 прогон автотестов + скриншоты.
